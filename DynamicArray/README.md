# Dynamic Array

## 정의
일반 배열은 생성 시 크기를 고정해야 하므로, 저장할 원소 수를 미리 알 수 없는 경우 크기를 넉넉히 잡거나 배열이 가득 찼을 때 새로운 배열을 만들어 데이터를 옮겨야 한다. 동적 배열은 이 과정을 내부적으로 자동화하여, 크기 제약 없이 사용할 수 있도록 만든 자료구조이다.

## 파일 구성
- `dynamicArray.h`: 동적배열 구현에 필요한 구조체 및 함수 정의
- `dynamicArray.c`: 배열 생성, 파괴, 원소 삽입, 삭제 등의 함수 구현
- `dynamicArrayMain.c`: 메인함수

## 동작 방식
### 크기관리
- 생성 시 초기 크기만큼 메모리 할당
- 가득 차면 크기를 두배로 늘리고, `realloc()`으로 메모리 재할당
### 지원함수
- 배열의 맨 끝에 원소를 삽입하는 push_back, 맨 끝의 원소를 삭제하는 pop_back 함수
- 배열의 원하는 인덱스에 원소를 삽입하는 insert, 원하는 인덱스의 원소를 삭제하는 erase 함수

## 빌드 및 실행
```bash
make
./dynamicArrayMain
```

## 시간 복잡도
| 연산 | 복잡도 | 설명 |
|------|--------|------|
| push_back / pop_back | O(1) (amortized) | 두배씩 확장하여 realloc 빈도를 줄임 |
| insert / erase | O(n) | 이후 원소들을 한 칸씩 이동해야 함 |
