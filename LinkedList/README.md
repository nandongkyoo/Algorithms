# Linked List

## 정의
연결 리스트는 각 노드가 데이터와 다음 노드를 가리키는 포인터를 함께 저장하여, 메모리상 흩어져 있어도 포인터로 노드들을 연결하여 관리하는 자료구조이다. 인덱스 접근은 순회가 필요하기 때문에 배열보다 느리지만, 삽입/삭제는 배열보다 빠르다는 장점이 있다.

## 파일 구성
- `linkedList.h`: 연결 리스트 구현에 필요한 구조체 및 함수 정의
- `linkedList.c`: 리스트 생성, 파괴, 삽입, 삭제 등의 함수 구현
- `linkedListMain.c`: 메인함수

## 동작 방식
### 더미 노드
- 더미노드가 없으면 `head`가 `NULL`일 수 있어서, 함수마다 리스트가 비어있는지를 분기 처리해야한다. 그러나 더미노드를 두면 head가 항상 유효한 노드를 가리키기 때문에, "삽입할 위치의 이전 노드"가 항상 존재한다는 불변식이 성립한다.

### head / tail
- `head`는 더미노드를 `tail`은 마지막 노드를 가리키는 포인터 변수로 노드를 처음이나 마지막에 삽입 또는 삭제를 O(1)에 가능하도록 한다.

### 지원함수
- 리스트 맨 앞/뒤에 원소를 삽입하는 `push_front/push_back`, 맨 앞/뒤 원소를 삭제하는 `pop_front/pop_back` 함수
- 원하는 인덱스에 원소를 삽입하는 `insert`,  원하는 인덱스의 원소를 삭제하는 `erase` 함수
- 특정 값을 가진 노드를 찾는 `find` 함수
- 리스트의 모든 노드를 지우지만 리스트 자체는 재사용이 가능한 상태로 남겨놓는 `clear`, 리스트를 완전히 해제하는 `destroy` 함수
- 리스트가 비어있는지 확인하는 `is_empty`, 원소 개수를 반환하는 `size` 함수

## 빌드 및 실행
```bash
make
./linkedListMain
```

## 시간 복잡도
| 연산 | 복잡도 | 설명 |
|------|--------|------|
| push_front | O(1) | 더미 바로 뒤의 노드만 확인하기 때문에 순회 없이 O(1) |
| push_back | O(1) | `tail` 포인터로 바로 연결하기 때문에 O(1) |
| pop_front | O(1) | 더미 바로 뒤의 노드만 확인하기 때문에 순회 없이 O(1) |
| pop_back | O(n) | 싱글 연결 리스트이기 때문에 tail 이전의 노드를 알 수 없어 `head`부터 순회해야 함 |
| insert / erase | O(n) | 인덱스가 0이나 맨 끝인 특수한 경우는 O(1)이지만, 일반적인 경우에는 순회를 해야하기 때문에 O(n) |
| find | O(n) | 값 하나하나를 비교하며 순회하므로 O(n) |
